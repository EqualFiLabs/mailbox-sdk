# Versioning Policy

This repository follows **Semantic Versioning**.

- `MAJOR`: breaking API changes
- `MINOR`: backwards-compatible feature additions
- `PATCH`: backwards-compatible fixes

Current release target: **0.2.0**

Guidance:
- treat exported `Mailbox` methods as public API
- document API changes in README and release notes
- use conventional commit prefixes (`feat:`, `fix:`, `chore:`) to keep release intent clear
